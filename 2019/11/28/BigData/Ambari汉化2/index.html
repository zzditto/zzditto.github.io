全部步骤简化

  1. 汉化messages.js和i18n.config.js
  2. 将汉化完成的ambari2.6.2源码上传到虚拟机test上，解压
  3. 跳转到对应目录ambari-web
  4. 控制台页面的编译使用brunch buld编译，平台管理页面编译使用gulp编译
  5. 编译完成之后，将整个源码文件打包scp到ambari主服务器上，建立软连接，重启ambari-server即可

注：目前已经完成编译之前的软件和依赖包的安装，将test虚拟机上的整个源码包打包，下载，后续只需要替换js文件，再执行编译命令即可【就是说，这个编译后的源码用到老】
